RETRYABLE READ
Automatically retries a read operation if transient network error occurs.

db.orders.find({ status: "pending" }, { retryReads: true });

Useful for distributed systems.

===========================================================

RETRYABLE WRITE
Ensures a write operation is **idempotent** and retried automatically in case of transient errors.

db.orders.updateOne(
  { orderId: 100 },
  { $set: { status: "shipped" } },
  { retryWrites: true }
)
